General-purpose hash functions (MD5, SHA-1, SHA-256, SHA-512) must not be used directly to hash passwords. They are designed to be fast — an attacker can compute billions of hashes per second with commodity hardware, making brute-force and rainbow table attacks trivial.

Passwords must be hashed with a purpose-built password hashing function that includes a work factor: Argon2, bcrypt, scrypt, or PBKDF2 with sufficient iterations.

PBKDF2 iteration threshold: 600,000 (OWASP 2025 recommendation for PBKDF2-HMAC-SHA256).

Noncompliant Code Example

// Noncompliant — SHA-256 is fast; not suitable for passwords
byte[] hash = MessageDigest.getInstance("SHA-256").digest(password.getBytes()); // Noncompliant

// Noncompliant — PBKDF2 iteration count too low
PBEKeySpec spec = new PBEKeySpec(password, salt, 10_000, 256); // Noncompliant

// Noncompliant — no salt
byte[] hash = MessageDigest.getInstance("SHA-512").digest(password.getBytes()); // Noncompliant

Compliant Solution

// Argon2 (recommended)
Argon2PasswordEncoder encoder = Argon2PasswordEncoder.defaultsForSpringSecurity_v5_8();
String encoded = encoder.encode(rawPassword);

// PBKDF2 with sufficient iterations
PBEKeySpec spec = new PBEKeySpec(password, salt, 600_000, 256); // Compliant

// bcrypt (work factor 12+)
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder(12);
String encoded = encoder.encode(rawPassword);
