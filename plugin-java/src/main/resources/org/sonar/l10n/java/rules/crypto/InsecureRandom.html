java.util.Random and Math.random() are not cryptographically secure pseudo-random number generators (CSPRNGs). Their output is predictable and must not be used for security-sensitive values such as tokens, session IDs, keys, salts, or nonces.

SecureRandom seeded with a fixed value is also flagged — a fixed seed produces deterministic output, defeating the purpose of secure randomness entirely.

Noncompliant Code Example

// Noncompliant — Random used for security token
String sessionToken = Long.toHexString(new Random().nextLong()); // Noncompliant

// Noncompliant — Math.random() for key material
byte[] salt = BigInteger.valueOf((long)(Math.random() * Long.MAX_VALUE)) // Noncompliant
    .toByteArray();

// Noncompliant — fixed seed makes SecureRandom deterministic
SecureRandom sr = new SecureRandom(new byte[]{1, 2, 3}); // Noncompliant

Compliant Solution

// Use SecureRandom without a fixed seed
SecureRandom sr = new SecureRandom();
byte[] token = new byte[32];
sr.nextBytes(token);
